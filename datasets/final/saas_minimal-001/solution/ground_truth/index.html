02 — Query plane

Ask in SQL. Answer in 38ms.

RiverQL is PostgreSQL on the outside, columnar trace storage on the inside. Group by route, filter by attribute, join with logs — the way you already think.

  • Sub-second queries across 30 days of retention
  • Saved views become Grafana panels and PagerDuty rules
  • Native joins between spans, logs and deploy markers
  • BYO S3 for cold tier — keep raw spans for a year
-- p95 latency on /api/checkout, by region, last 1h
SELECT
  attrs.region,
  percentile(duration_ms, 0.95) AS p95,
  count(*) AS requests
FROM spans
WHERE name = 'POST /api/checkout'
  AND ts > now() - interval '1h'
GROUP BY attrs.region
ORDER BY p95 DESC;
