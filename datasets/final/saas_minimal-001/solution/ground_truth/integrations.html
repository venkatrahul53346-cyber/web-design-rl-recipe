Install in 30 seconds

One command, every service.

The agent attaches at process start. Restart your service and the first trace shows up before the page refreshes.

Quickstart guide→ View on GitHub
01 — Node.js
$ npm install @riverbed/agent
$ node -r @riverbed/agent server.js
02 — Python
$ pip install riverbed-agent
$ riverbed-run -- gunicorn app:wsgi
03 — Go
$ go install riverbed.dev/cmd/riverbed@latest
$ riverbed build -o bin/api ./cmd/api
04 — Kubernetes
$ kubectl apply -f https://get.riverbed.dev/k8s.yaml
Standards & interop

OpenTelemetry first

Already running OTel SDKs? Riverbed accepts your existing OTLP stream — point it at our endpoint and you’re live. No re-instrumentation.

W3C trace context

Standard traceparent and tracestate headers, so context survives whatever proxies and SDKs sit between your services.

Self-host or hosted

The query plane runs as a single binary against Postgres + S3. Run it on your VPC, or let us run it. The agent doesn’t care.
