What you get

Three things that used to take a quarter, now take an afternoon.

No code changes. No sidecar contortions. No vendor agent rewriting your binaries at runtime.

01 / Capture

Zero-instrumentation tracing

The Riverbed agent attaches at the OS layer using eBPF. It reads syscalls and TLS metadata, then reconstructs spans for HTTP, gRPC, Postgres, Redis, Kafka, and 40+ other protocols. Your services don't import anything.

eBPF · linux 5.8+ · < 1.4% CPU overhead
02 / Query

TraceQL, not dashboards

Filter by any tag, latency bucket, error class, or causal path. Save queries as alerts. Pin them to a service. The query plane is hosted — you don't run another database.

p99 query < 800ms across 14d retention
03 / Keep

Tail-based sampling that's actually honest

Every trace is scored after the fact: errors, slow tails, and rare paths are retained at 100%. Healthy traffic is downsampled. You pay for signal, not noise.

keep 0.2% of volume · catch 100% of incidents
How it fits

A small agent on every node. A hosted plane you never page on-call for.

The split is intentional. Capture stays inside your perimeter and runs as a daemonset. Querying, retention, and alerting are managed by us — so storage scaling never lands in your sprint.

  1. 01riverbed-agentOpen source, Apache-2.0. Ships traces over a single mTLS connection. Buffers locally when the plane is unreachable.
  2. 02Hosted query planeMulti-tenant ClickHouse with row-level isolation. SOC 2 Type II, regional residency in US, EU, AP.
  3. 03OpenTelemetry-compatibleAlready emitting OTLP? Point your collector at our endpoint and skip the agent. Same TraceQL, same plane.
