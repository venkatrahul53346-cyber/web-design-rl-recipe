# Web Design Replication Task

You are given screenshots of a 6-page website at `/app/prompt/screenshots/`:
- `/app/prompt/screenshots/faq.png`
- `/app/prompt/screenshots/index.png`
- `/app/prompt/screenshots/preorder.png`
- `/app/prompt/screenshots/press.png`
- `/app/prompt/screenshots/specs.png`
- `/app/prompt/screenshots/story.png`

Recreate the design as faithfully as possible using HTML and CSS.

## Required output files (write them to `/app/`)
- faq.html
- index.html
- preorder.html
- press.html
- specs.html
- story.html
- styles.css

## Available placeholder assets

The following files are pre-staged in `/app/`. Reference them from your HTML if the design has images:
- `/app/illustration-abstract.jpg` — use as `<img src="illustration-abstract.jpg">` in your HTML
- `/app/photo-landscape-1.jpg` — use as `<img src="photo-landscape-1.jpg">` in your HTML
- `/app/photo-portrait-1.jpg` — use as `<img src="photo-portrait-1.jpg">` in your HTML
- `/app/photo-product-1.jpg` — use as `<img src="photo-product-1.jpg">` in your HTML
- `/app/photo-product-2.jpg` — use as `<img src="photo-product-2.jpg">` in your HTML

## Rules
1. Each `.html` file must `<link rel="stylesheet" href="styles.css">`. Put all styling in `styles.css`.
2. Match colours, spacing, typography, and layout structure as closely as you can.
3. Functionality is out of scope; visual fidelity is what is graded.
4. Do not modify or read files under `/tests/` or `/solution/`.
