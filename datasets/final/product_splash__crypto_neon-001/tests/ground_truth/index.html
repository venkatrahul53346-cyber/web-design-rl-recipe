01 · ANATOMY OF A REVIEW

It comments
like the engineer
you wish you had.

No "consider refactoring this for clarity." Pliny writes the suggestion in code, cites the file it would regress, and tells you when to ignore it.

A.

Reasoned, not pattern-matched

Every comment links the upstream caller, the test that would fail, and the assumption it violates.

B.

Suggestions as commits

Accept with a single click. Pliny opens a fixup commit against your branch — never force-pushes, never rewrites history.

C.

Severity it can defend

Blocking, advisory, or nit — graded against your team's prior reviews, not a generic rubric.

02 · WHAT IT ACTUALLY CATCHES

The six classes
of bug that pass
human review.

Linters catch typos. Static analyzers catch nullability. Pliny is calibrated for the bugs that ship to prod because three reviewers all assumed someone else had read the other file.

CLASS · 01

Cross-file invariants

A constant changed in one module that quietly breaks an assumption in another. The kind of bug a 400-line diff hides behind a thousand benign ones.

e.g. enum value renamed, switch un-exhaustive
CLASS · 02

Concurrency regressions

Races that only fire under load. Pliny re-runs the diff against a model of your runtime, not just the AST.

e.g. dropped lock, missing await, lost write
CLASS · 03

Migration footguns

Schema changes that lock prod tables, break replicas, or strand readers on the old version. Cross-referenced with your deploy history.

e.g. ALTER without CONCURRENTLY, NOT NULL backfill
CLASS · 04

Silent contract changes

Renaming a JSON field, changing a return shape, narrowing a type — the kind of break that compiles fine and pages your downstream team at 4am.

e.g. response field removed, public API drift
CLASS · 05

Test-shaped holes

New branches without coverage; assertions weakened to make CI green; mocks that hide the real failure mode.

e.g. assertTrue(true), commented-out test
CLASS · 06

Security & secrets

SSRF surfaces, auth bypasses, IDOR patterns, and the API token someone pasted into a fixture file at 11:48pm on a Friday.

e.g. unscoped query, hardcoded JWT, open redirect
