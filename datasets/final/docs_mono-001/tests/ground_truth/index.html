01 · the pipeline

From openapi.yaml to honest, tested, typed code — five stages, one command.

001

Resolve

Inline $ref, validate against OpenAPI 3.0/3.1, surface ambiguities before they become bugs.

002

Type

Emit ergonomic clients in TypeScript, Go, Python, Rust, Kotlin, Swift — with discriminated unions, not any.

003

Mock

Spin up a stateful mock server in 200ms. Faker rules, scenario files, latency injection, examples replayed verbatim.

004

Verify

Replay 100+ generated cases against any environment. Fails the build the moment the implementation drifts.

005

Diff

Semantic diffing classifies each change as breaking, additive, or patch. Block PRs accordingly.

02 · why bother

Most API drift is invisible until production.

OpenAPI specs rot the moment they leave review. The frontend team trusts a doc that doesn't match the gateway. The backend ships a "non-breaking" rename that silently nulls a field downstream. Pact files go stale. Mocks lie.

Glyph fixes the loop by making the spec the single source of truth — and by failing loudly the second the running service stops honoring it. No more "works on the docs site".

If glyph contract verify passes in CI, your producer and your generated clients agree on every field, every enum, every error shape. That's the whole product.

Without contracts

  • Spec is updated after the endpoint ships.
  • Frontend types are hand-written, then forgotten.
  • Mock server returns whatever the author remembered.
  • Breaking changes ship under a "chore:" commit.
  • Integration bugs land in staging, not in review.
  • Postman collections diverge from production.

With glyph

  • Spec change is the first commit in every PR.
  • Clients regenerate on postinstall; the type-checker catches drift.
  • Mock is derived from the spec — examples are honored verbatim.
  • glyph diff labels breakage; CI refuses to merge it silently.
  • Contract suite runs against staging on every deploy.
  • One artifact — openapi.yaml — drives every consumer.
03 · a closer look

A generated client reads like one you'd write by hand.

openapi.yamlyaml
paths:
  /invoices/{id}:
    get:
      operationId: getInvoice
      parameters:
        - in: path
          name: id
          schema: { type: string, format: uuid }
      responses:
        '200':
          content:
            application/json:
              schema: { $ref: '#/components/schemas/Invoice' }
        '404': { $ref: '#/components/responses/NotFound' }
src/glyph/billing.tsgenerated · typescript
// AUTOGENERATED by glyph v0.14.2 — do not edit.
import { Client, Result } from "./runtime";
import { Invoice, NotFound } from "./schemas";

export async function getInvoice(
  client: Client,
  id: string,
): Promise<Result<Invoice, NotFound>> {
  return client.request({
    method: "GET",
    path: `/invoices/${id}`,
    expects: { 200: Invoice, 404: NotFound },
  });
}

Discriminated Result<Ok, Err> types force consumers to handle every documented error. No silent catch. No any creeping in from oneOf.
