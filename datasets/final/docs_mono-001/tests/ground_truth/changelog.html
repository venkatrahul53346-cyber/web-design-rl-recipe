Changelog

Releases

Glyph follows semantic versioning. While we are pre-1.0, minor releases (0.x) may contain breaking changes — these are always flagged and accompanied by codemods where possible.

Subscribe via RSS, or watch the GitHub releases feed. Older entries (pre-0.10) live in the archive.

v0.14.2
2024-11-04
patch
17 commits · 4 contributors

Fixed

  • glyph generate no longer panics when a oneOf branch is itself a $ref to a discriminator-tagged schema. #412
  • TypeScript client: response bodies typed as never when a 204 was the only documented status. Now correctly emit void. #418
  • Mock server respects x-glyph-seed on nested objects, producing stable fixtures across runs. #421
  • Resolved a race in glyph watch where two near-simultaneous spec writes would emit only one regeneration. #425

Changed

  • Contract test diffs now collapse identical array elements past index 3, controlled by --diff-elide.
  • Bumped @apidevtools/json-schema-ref-parser to 11.7.0.
v0.14.0
2024-10-22
breaking
94 commits · 11 contributors

Introduces the producer contract workflow: Glyph can now record real responses from a running service and diff them against the OpenAPI spec, failing CI when the implementation drifts from the contract.

Migration: the glyph test subcommand has been split into glyph test consumer and glyph test producer. Run npx glyph codemod 0.14 to update your config and CI scripts.

Added

  • New command glyph record — captures HTTP traffic via a local proxy and writes a producer contract bundle. #367
  • Producer contract verification: glyph test producer --against build/contract.json.
  • Support for OpenAPI 3.1 patternProperties in both TS and Go generators.
  • glyph init now scaffolds a glyph.config.ts with typed config, replacing the previous YAML.

Breaking

  • glyph test alone is no longer valid. Use the consumer/producer subcommands.
  • The legacy --ts-strict flag has been removed; strictness now follows your project's tsconfig.json.
  • Generated client method names switched from snake_case to the spec's operationId verbatim. Provide --rename camelCase to restore previous behavior.
v0.13.4
2024-09-30
security
8 commits · 3 contributors

Security

  • GHSA-7w8x-mpvh-q92f — the mock server's CORS preflight handler would echo back arbitrary Origin values when --cors-permissive was set. The flag is now ignored unless NODE_ENV=development, and logs a warning. Reported by @nadiakem.
  • Upgraded transitive ws to 8.18.0 to address advisory GHSA-3h5v-q93c-6h2m.

Fixed

  • Stack overflow on circular schemas where $ref pointed through an allOf back to the parent. #341
v0.13.0
2024-09-09
minor
62 commits · 9 contributors

The Go generator graduates from experimental. Output now mirrors the ergonomic choices of the TypeScript generator: tagged unions for oneOf, typed errors per response status, and a WithContext variant for every operation.

payments_client.gogo
func (c *Client) CreateCharge(ctx context.Context, body CreateChargeBody) (CreateChargeResponse, error) {
    // 200, 402, 422 are typed as distinct branches of the response union
    return c.do(ctx, "POST", "/v1/charges", body)
}

Added

  • Go generator: --lang go --module github.com/acme/api is now stable.
  • x-glyph-deprecated-after vendor extension — emit a build warning if the current date is past the listed value.
  • Plug-in API for custom templates. See guides/plugins.

Changed

  • Diff output uses unified hunks; legacy side-by-side mode available via --diff-style=side.
  • Default mock latency lowered from 120ms to 0ms. Restore previous behavior with mock.latency = "realistic" in config.
v0.12.1
2024-08-19
patch
21 commits · 6 contributors

Fixed

  • Windows: paths containing spaces in --out no longer truncate at the first space. #298
  • glyph lint reported a false positive on valid nullable: true + enum combinations under OpenAPI 3.0.3. #301
  • Restored compatibility with Node 18.17 — accidentally relied on structuredClone behavior introduced in 18.20. #305

Docs

  • Added a monorepo guide covering pnpm workspaces and Turborepo caching.
v0.12.0
2024-07-30
breaking
88 commits · 14 contributors

Schema resolution rewritten to be incremental. On a typical 4,000-operation spec, glyph generate cold-start time drops from 11.4s to 2.1s; glyph watch re-emits in under 300ms for single-path edits.

Added

  • Incremental resolver with on-disk cache at .glyph/cache. Disable with --no-cache.
  • glyph explain <operationId> — prints the full resolved schema graph for one operation, useful for debugging discriminators.

Breaking

  • Spec files must now be UTF-8. UTF-16 BOM-prefixed YAML is rejected with a clear error rather than silently mis-parsed.
  • The --legacy-anyof compatibility flag, deprecated in 0.10, has been removed.
v0.11.2
2024-06-25
patch
12 commits · 5 contributors

Fixed

  • Contract test runner now exits with code 2 on spec/implementation drift (was 1, indistinguishable from a runner crash). #251
  • Mock server: format: date-time fixtures occasionally produced invalid ISO strings on the last day of a month. #256
  • glyph diff truncated change summaries past 50 lines; the limit is now configurable and defaults to unlimited in CI.
v0.11.0
2024-05-14
minor
71 commits · 10 contributors

Mocks become first-class. The previous glyph serve command is replaced by glyph mock, which speaks both HTTP and a programmatic in-process API for test harnesses.

Added

  • glyph mock with deterministic fixtures driven by example, examples, and the new x-glyph-seed extension.
  • Programmatic mock: import { mock } from "@glyph/mock" for Vitest, Jest, and Playwright integration.
  • Lint rule no-untagged-oneof — requires a discriminator on every oneOf with three or more branches.

Deprecated

  • glyph serve prints a deprecation notice and forwards to glyph mock. It will be removed in 0.14.

Earlier releases

Versions 0.1 through 0.10 are kept for reference but receive no further patches. Read them in the archived CHANGELOG-0.x.md on GitHub, or browse the tag list.

If you are upgrading across a major gap, the upgrade guide documents the codemods you should run in order, and what to expect at each step.
