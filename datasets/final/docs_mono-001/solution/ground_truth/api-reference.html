CLI reference · v0.14.2

Command-line interface

The glyph binary is the single entry point. Each subcommand reads glyph.toml by default and emits stable, machine-readable output on stderr when --json is set. Flags shown with --no-* negate booleans defined in config.

glyph generate

Reads an OpenAPI document and emits typed clients for one or more targets. Generation is deterministic: identical input plus identical config produces byte-identical output, which is what makes the verify --frozen check in CI meaningful.

since 0.1.0 stable yes side effects writes to out_dir

Signature

$ glyph generate <spec> --target <ts|go|py|rust> --out <dir> [--profile <name>] [--frozen] [--json]

Parameters

NameTypeDescription
spec path | url required Path or URL to the OpenAPI 3.0 / 3.1 document. .yaml, .yml, and .json are auto-detected. Refs are resolved relative to the document root.
--target enum required Language target. May be repeated to fan out generation in a single pass (e.g. --target ts --target go).
--out path required Output directory. Glyph treats this as owned: files not in the generation manifest are pruned on each run.
--profile string optional Named profile from [profiles.*] in glyph.toml. Defaults to default.
--frozen boolean optional Fails with exit code 17 if any output file would change. Use in CI to assert checked-in clients match the spec.
--json boolean optional Emit a structured generation manifest on stdout. Human progress remains on stderr.

Request — invocation

shellbash
$ glyph generate openapi.yaml \
    --target ts \
    --out ./packages/client \
    --profile strict \
    --frozen --json

Response — manifest on stdout

stdoutjson
{
  "glyph": "0.14.2",
  "target": "ts",
  "profile": "strict",
  "spec_digest": "sha256:7f3a…b21c",
  "operations": 48,
  "schemas": 112,
  "files": [
    { "path": "client.ts",        "bytes": 8421,  "changed": false },
    { "path": "models/index.ts", "bytes": 26104, "changed": false },
    { "path": "runtime.ts",       "bytes": 3902,  "changed": false }
  ],
  "frozen_ok": true,
  "elapsed_ms": 214
}
Note When --frozen trips, the manifest still emits but with frozen_ok: false and a diff array listing the offending files. Pipe through jq '.diff' for a one-glance CI failure.

glyph mock

Starts a local HTTP server that responds to every operation described in the spec using the operation's example, schema examples, or schema-derived synthetic data — in that order. The server is request-validated: malformed bodies, missing required headers, or out-of-range parameters return 400 with the failing JSON Pointer, never a generic 500.

since 0.4.0 stable yes side effects binds TCP port

Signature

$ glyph mock <spec> [--port <n>] [--seed <n>] [--latency <ms>] [--scenario <file>] [--strict-headers]
returns long-running process; SIGINT shuts down cleanly with a summary of unmatched requests on stderr.

Parameters

NameTypeDescription
--port integer optional TCP port to bind. Default 4010. Pass 0 to bind to any free port (printed on startup).
--seed integer optional Deterministic seed for the synthetic-data generator. Same seed + same spec + same request ⇒ same response body.
--latency integer | range optional Injected response delay in ms. Accepts a range (50..300) to simulate jitter.
--scenario path optional YAML file mapping request matchers to canned responses. Useful for testing error paths the spec doesn't surface as defaults.
--strict-headers boolean optional Reject requests whose Content-Type or Accept aren't explicitly declared on the operation. Off by default.

Request — call into the mock

shellbash
$ glyph mock openapi.yaml --port 4010 --seed 42 &
# glyph mock · listening on http://127.0.0.1:4010 · seed=42

$ curl -sS http://127.0.0.1:4010/v1/invoices/inv_8821 \
       -H 'Accept: application/json'

Response — synthesized from schema

200 OK · application/jsonjson
{
  "id": "inv_8821",
  "status": "open",
  "customer": {
    "id": "cus_3a91",
    "email": "a.lindgren@example.com"
  },
  "line_items": [
    { "sku": "SEAT-ANN", "qty": 3, "unit_cents": 12000 }
  ],
  "subtotal_cents": 36000,
  "currency": "USD",
  "created_at": "2024-11-04T14:08:11Z"
}

glyph verify

Replays recorded traffic — or live integration tests — against the spec and reports any drift. Each interaction is checked in both directions: the request must be valid against the operation, and the response must satisfy the declared status, headers, and schema. Drift is the only thing it reports; matches are silent unless --verbose.

since 0.7.0 stable yes side effects none (read-only)

Signature

$ glyph verify <spec> --recording <file> [--include <glob>] [--fail-on <level>] [--report <path>]

Parameters

NameTypeDescription
--recording path required HAR file or Glyph .jsonl trace produced by the runtime SDK's record() middleware.
--include glob optional Operation-id glob (e.g. invoices.*) to scope the check. Repeatable. Defaults to all operations.
--fail-on enum optional Severity threshold: error (default), warn, or info. Determines the process exit code.
--report path optional Write a JUnit-compatible XML report alongside the human summary. Used by the GitHub Action to annotate PRs.

Request — verify a captured trace

shellbash
$ glyph verify openapi.yaml \
    --recording ./traces/staging-2024-11-04.jsonl \
    --include 'invoices.*' \
    --fail-on error

Response — drift report on stderr

stderrtext
glyph verify · 1,204 interactions · 3 drifts

error  invoices.create        response.body
       at $.line_items[0].tax_rate
       expected: number (0..1)
       received: "0.0875"     # string, not number
       trace:    traces/staging-2024-11-04.jsonl:482

error  invoices.get           response.status
       expected: 200 | 404
       received: 418
       trace:    traces/staging-2024-11-04.jsonl:903

warn   invoices.list          response.header
       x-next-cursor declared but absent on 12/40 responses

exit 19 (drift detected)
