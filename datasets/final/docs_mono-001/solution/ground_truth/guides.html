Recipes for the long road between schema and shipped.

Practical, opinionated walkthroughs for the things that go wrong in real API work — drift between consumer and producer, breaking changes that slip past code review, mocks that lie, and CI that doesn't catch any of it. Each recipe is a focused workflow you can adopt in an afternoon.

★ Editor's pick

Catching breaking changes in pull requests, automatically.

Wire glyph diff into your PR checks so the bot comments with a structured changelog: which endpoints changed shape, which response codes were removed, and which fields became required. Producer teams stop guessing what consumers will break.

Includes a GitHub Action, a fail-on-major policy file, and a worked example of an additionalProperties: false tightening caught at review time.

read the recipe → 
$ glyph diff main HEAD --policy semver

# 3 changes detected against main
  major  POST /orders        response.shipping_address now required
  minor  GET  /orders/{id}   added field: tracking_url (nullable)
  patch  GET  /healthz       description updated

✖ policy violation: 1 major change on protected branch.
  → blocking PR #4821. require label "api:breaking" to override.
